SevNTU-project by APVS-async-modeling

Asynchronous modeling education student project.

1). «Общий интерфейс» Графический интерфейс программы (использует модуль 7 для вывода состояния системы) + соединение всех остальных модулей. (А) (Ярославцев) 2). «Создание модели ДУ» = создание модели входной схемы. Этот модуль читает входной файл описания ДУ и создает модель всего устройства в памяти. (Б) (Данильчук) 3). «Создание модели элементов ДУ». Все, кто занимается этой частью, вместе разрабатывают общий формат задания одного элемента в файле-библиотеке элементов. Т.е. - описание элемента унифицировано и его формат проектируется вместе всеми, у кого вариант «Г». Каждый, кто делает эту часть, должен написать две процедуры. Первая – создает модель элемента ДУ данного типа в памяти, вторая – задает функцию элемента данного типа при моделировании. Описание элемента находится в файле-библиотеке элементов. Этот модуль должен найти заданную модель в библиотеке и оттранслировать. (Г) (Доля, Махмудов, Кекух, Грохольский) 4). Моделирование ДУ. Выполняет моделирование на уровне схемы. Этот модуль считывает тестовые наборы из файла, содержащего значения входных сигналов схемы, выводит результат моделирования в файл протокола моделирования, для выполнения моделирования элемент обращается к функциям модулей «Г». (В) (Григоров) 5). Интерфейс вывода временных диаграмм. Берет результаты моделирования из файла-протокола отладки, запрашивает у пользователя список контактов, для которых должны быть постоены диаграммы, и должен их построить в отдельном окне. Дополнительно: при наличии пошаговой обработки – м. быть возможно пошаговое построение временных диаграмм. (Д) (Середа) 6). Модуль анализа результатов моделирования. Должен реализовать проверку некоторого набора опасных состояний (риски) по результатам моделирования. На входе модуля - файл-протокола отладки. На выходе – список рисков и других опасных ситуаций с описанием их типов, записанный в текстовый файл – протокол анализа результатов. Должна быть задана возможность проверять ситуации, и создавать протокол. Для заданного набора – проверить риск, временной интервал считать таким-то, таким-то. Составить протокол – главное. В такое-то время – такой-то риск и т.п. (Е) (Монастырный) 7). Модуль (интерфейс) вывода состояния системы. Обращается к модулю моделирования (В) и выводит его состояние в графический интерфейс пользователя (для каждого шага или каждого тестового набора, точнее, по требованию модуля моделирования). Т.е. – в данном модуле должна быть функция, которая по требованию извне обращается к модулю моделирования и читает из него значения внутренних рабочих структур, а затем – выводит считанные значения в графический интерфейс в удобочитаемом форматированном виде. (Ж) (Глазырин)
